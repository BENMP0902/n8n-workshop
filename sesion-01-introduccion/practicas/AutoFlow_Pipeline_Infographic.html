Audit Logger
code_node + db_node
Recibe datos de TODOS los caminos (pass, reject, block). Registra: timestamp, datos originales, cada validación con resultado, estado final, ejecutivo asignado (si aplica). Almacena en base de datos o Google Sheet como audit trail.

R-010 logging obligatorio NO almacenar passwords/tokens PII: least privilege access

🔒 Security-First Analysis

Input Validation

Sanitizar TODOS los campos antes de procesar. Prevenir XSS, SQL injection y buffer overflow limitando longitud.

Rate Limiting

Máx 100 req/min por IP en el webhook. Sin esto, un atacante puede ejecutar un DDoS llenando el sistema de leads falsos.

Webhook Authentication

Verificar HMAC signature o shared secret en headers. Previene spoofing de leads.

Data Privacy (PII)

Principio de least privilege: cada nodo accede solo a lo necesario. Credenciales en credential store, nunca en nodos.
